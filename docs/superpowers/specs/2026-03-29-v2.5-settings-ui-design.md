# v2.5 Design: Unified Settings / Onboarding UI

## Summary

Replace the separate onboarding wizard with a single, reusable Settings window that serves both first-run onboarding and ongoing preferences. Accessible from a new "Preferences..." menu item in the status bar menu.

## Goals

- One UI to learn, one UI to maintain
- Add language selection (full whisper language list, no auto-detect)
- Add hotkey recorder (press-to-record with Cancel/Confirm flow)
- Preferences window is pre-populated from current config

## Non-Goals

- Model size selection (stays as config-file-only for now)
- Type mode toggle (v3+)
- Sound feedback toggle (config-file-only)

---

## Window Layout

Same Cocoa NSWindow as current onboarding. Two modes controlled by a `isOnboarding` flag passed from Go.

**Window**: 480 x 580px, titled "JoiceTyper Setup" (onboarding) or "JoiceTyper Preferences" (prefs).

### Steps

| # | Section | Onboarding | Preferences |
|---|---------|------------|-------------|
| 1 | Accessibility | Live polling + prompt | Read-only indicator |
| 2 | Input Monitoring | Live polling + prompt | Read-only indicator |
| 3 | Select Microphone | Dropdown, default selected | Dropdown, current config selected |
| 4 | Language | Dropdown (100 languages), default "English" | Dropdown, current config selected |
| 5 | Hotkey | Shows default "Fn + Shift", "Change" button | Shows current config, "Change" button |
| 6 | Download Model | Progress bar | Hidden if model exists |
| 7 | Ready / Save | "Start JoiceTyper" button | "Save" button |

### Hotkey Recorder Flow

Default state:
```
Hotkey: [Fn + Shift]  [Change]
```

After clicking "Change":
```
Hotkey: [Press keys...]  [Cancel] [Confirm]
```

- The label becomes a live display of currently held modifier flags
- Monitors `kCGEventFlagsChanged` via an event tap (same mechanism as the main hotkey listener)
- Only modifier keys accepted (Fn, Shift, Ctrl, Option, Cmd) — consistent with existing `validKeys` in config.go
- "Cancel" reverts to previous value, "Confirm" locks in the new combo
- Must have at least one modifier selected to confirm

### Language Dropdown

Full list of 100 whisper-supported languages, displayed as:
```
English
Chinese
German
Spanish
...
Cantonese
```

Sorted by whisper's internal ID order (same order as `g_lang` in whisper.cpp). The dropdown stores the language code (`en`, `zh`, `de`, ...) and displays the full name with first letter capitalized.

Default selection: English (`en`).

### Data Flow

**Onboarding mode** (first run):
```
main.go → RunSetupWizard() → showSettingsWindow(1) → [NSApp runModalForWindow:]
  → user completes steps → continueClicked → writes config → returns to main.go
```

**Preferences mode** (from menu bar):
```
statusbar click → preferencesClicked() → OpenPreferences()
  → showSettingsWindow(0) → window opens (non-modal, non-blocking)
  → user edits → "Save" → writes config → restarts hotkey listener with new keys
```

### Hotkey Recorder Implementation

- Create a temporary CGEvent tap when "Change" is clicked
- Monitor `kCGEventFlagsChanged` events
- Display held modifiers in real-time as text ("Fn + Shift + Cmd")
- On "Confirm": store the selected keys, destroy the temporary tap
- On "Cancel": revert display, destroy the temporary tap
- Uses the same `keyToFlag` / `flagToKey` mapping as the main hotkey system

### Config Write on Save (Preferences mode)

1. Read current config
2. Update `trigger_key`, `language`, `input_device` from UI values
3. Write YAML back to `~/Library/Application Support/JoiceTyper/config.yaml`
4. Signal main app to reload — stop current hotkey listener, re-create with new config

### Hotkey Restart on Change

When the user saves new hotkey settings from Preferences:
1. Call `hotkey.Stop()` — this calls `stopMainLoop()` via dispatch_async, unblocking `[NSApp run]`
2. `hotkey.Start()` returns in `runAppMode()`
3. The main loop needs to support re-entering `hotkey.Start()` with new config

This requires a structural change to `runAppMode()`: wrap `hotkey.Start()` in a loop that re-reads config and re-creates the listener when signaled.

## Config Changes

No new fields. Existing fields used:

```yaml
trigger_key:
  - fn
  - shift
language: en        # currently allows "" (auto-detect), will default to "en"
input_device: ""    # device name from PortAudio
```

### Validation Update

- `language`: validate against the full 100-language list (currently validates against a smaller set in config.go)
- `trigger_key`: no changes — already validates against `validKeys`

## Testing

- Existing unit tests unchanged (they use mock interfaces)
- Manual testing checklist:
  - [ ] First run: onboarding shows all steps, language defaults to English
  - [ ] First run: hotkey recorder works, saved to config
  - [ ] Subsequent run: no onboarding, hotkey works
  - [ ] Preferences: opens from menu bar, fields pre-populated
  - [ ] Preferences: change mic, language, hotkey → Save → config updated
  - [ ] Preferences: hotkey change takes effect without app restart
  - [ ] Preferences: Cancel reverts hotkey recorder
  - [ ] Reinstall: stale TCC entries cleared, permissions re-prompted
